    26‚Üí    26‚Üí## 1. What the original got right
    27‚Üí    27‚Üí
    28‚Üí    28‚ÜíBefore cataloguing failures, it is only fair to acknowledge the original's genuine strengths.
    29‚Üí    29‚ÜíA 48-hour hackathon that shipped a working end-to-end demo ‚Äî synthetic fleet simulator, live
    30‚Üí    30‚Üímap, operator console, alert workflow, and a chatbot ‚Äî is a real achievement. Specifically:
    31‚Üí    31‚Üí
    32‚Üí    32‚Üí- **Problem framing was excellent.** The "how full is the next jeepney?" gap is real,
    33‚Üí    33‚Üí  underserved, and aligns cleanly with UN SDG 9 and SDG 11. The dual-layer (edge + cloud)
    34‚Üí    34‚Üí  mental model is the correct architecture for this problem class.
    35‚Üí    35‚Üí- **The four-tier occupancy taxonomy** (üü¢ available / üü° filling / üî¥ at capacity / üî¥-blink
    36‚Üí    36‚Üí  overloaded) is intuitive, actionable, and maps cleanly to both a windshield LED and a map
    37‚Üí    37‚Üí  marker. This design choice survives into Re-LoadSense unchanged.
    38‚Üí    38‚Üí- **Operator-first alert verification** (ack ‚Üí verify ‚Üí false-alarm / escalate) is a
    39‚Üí    39‚Üí  thoughtful safety pattern that avoids black-box auto-escalation. It reflects genuine
    40‚Üí    40‚Üí  domain understanding of transit operations.
    41‚Üí    41‚Üí- **ASEAN expansion intent** ‚Äî seeding route data for PH / ID / MY / TH / VN and planning a
    42‚Üí    42‚Üí  GTFS import path ‚Äî showed the right ambition, even though the execution was uneven.
    43‚Üí    43‚Üí- **Contract-faithful simulation as a demo strategy** is, in principle, the right call for a
    44‚Üí    44‚Üí  hackathon: produce the same data shape a real deployment would emit, so downstream code is
    45‚Üí    45‚Üí  real even if the upstream is faked. (The execution of this principle was flawed ‚Äî see
    46‚Üí    46‚Üí  ¬ß4 ‚Äî but the principle is sound and Re-LoadSense retains it, honestly labeled.)
    47‚Üí    47‚Üí- **Config-driven thresholds** ‚Äî occupancy tiers, geofence radius, and capacity defaults
    48‚Üí    48‚Üí  live in `config/project_config.json` rather than being hardcoded. This is good practice
    49‚Üí    49‚Üí  and Re-LoadSense generalizes it into a full typed configuration system.
    50‚Üí    50‚Üí- **Parameterized SQL** ‚Äî `sqlite_store.py` uses parameterized queries throughout. No SQL
    51‚Üí    51‚Üí  injection vectors. This is a non-trivial discipline that many hackathon projects skip.
    52‚Üí    52‚Üí
    53‚Üí    53‚ÜíThese strengths are preserved in Re-LoadSense. Everything below is what must change.

    57‚Üí    57‚Üí## 2. Backend failures
    58‚Üí    58‚Üí
    59‚Üí    59‚ÜíThe backend (`backend/app/`) is a FastAPI app with ~740 lines of route handlers, a 1,208-line
    60‚Üí    60‚ÜíSQLite store, a 1,860-line god-module (`transit.py`), and five ‚Äî *five* ‚Äî chatbot files, two
    61‚Üí    61‚Üíof which are dead code.
    62‚Üí    62‚Üí
    63‚Üí    63‚Üí### 2.1 Five chatbot files, three of them dead
    64‚Üí    64‚Üí
    65‚Üí    65‚ÜíThe repo contains `chatbot.py` (Gemini), `no_API_chatbot.py` (heuristic), `ollama_chatbot.py`
    66‚Üí    66‚Üí(Ollama), plus `phase2.py` and `compat.py` tangled into the same concern. Grep confirms that
    67‚Üí    67‚Üí**`chatbot.py` and `ollama_chatbot.py` are never imported by the running application**. The
    68‚Üí    68‚Üílive code path (`state.py:FleetStore.recommendation()`) explicitly bypasses the LLM paths and
    69‚Üí    69‚Üícalls only `get_no_api_recommendation()`.
    70‚Üí    70‚Üí
    71‚Üí    71‚ÜíYet `requirements.txt` still ships `google-genai>=0.3.0` as a hard dependency, and the README
    72‚Üí    72‚Üíadvertises the Gemini chatbot as a feature. The Gemini file also contains a live bug
    73‚Üí    73‚Üí(`chatbot.py:69`) papered over with a comment that literally says *"wait, should be
    74‚Üí    74‚Üídestination string"* ‚Äî a developer's mid-fix note shipped to production.
    75‚Üí    75‚Üí
    76‚Üí    76‚Üí**Remediation:** One chatbot module with a strategy interface. Provider selection is a config
    77‚Üí    77‚Üíor feature-flag decision, not five copy-pasted files. Dead code is deleted, not archived.
    78‚Üí    78‚Üí
    79‚Üí    79‚Üí### 2.2 Zero authentication or authorization
    80‚Üí    80‚Üí
    81‚Üí    81‚ÜíNo endpoint has any auth. Anyone who can reach the API can:
    82‚Üí    82‚Üí
    83‚Üí    83‚Üí- `POST /api/database/reset` ‚Äî wipe everything (then auto-seeded with fake drivers).
    84‚Üí    84‚Üí- `POST /api/alerts/reset`, `POST /api/demand/reset` ‚Äî wipe alerts / demand artifacts.
    85‚Üí    85‚Üí- `DELETE /api/routes/{route}`, `DELETE /api/vehicles/{vehicle_id}` ‚Äî silent deletion.
    86‚Üí    86‚Üí- `POST /api/vehicles`, `POST /api/routes`, `POST /api/alerts` ‚Äî inject arbitrary data.
    87‚Üí    87‚Üí- Connect to `WS /ws/telemetry` and inject fake vehicle telemetry.
    88‚Üí    88‚Üí- `POST /api/chatbot` ‚Äî potentially expensive NLP/LLM calls.
    89‚Üí    89‚Üí
    90‚Üí    90‚ÜíThere are no API keys, no JWT, no OAuth, no role separation between commuters, operators, and
    91‚Üí    91‚Üíadmins. The static frontend is served from the same origin, so CSRF is also a concern for
    92‚Üí    92‚Üíevery state-changing endpoint.
    93‚Üí    93‚Üí
    94‚Üí    94‚Üí**Remediation:** Re-LoadSense ships auth in the first vertical slice. JWT access tokens +
    95‚Üí    95‚Üírefresh tokens, role-based access control (commuter / operator / admin / edge-device), per-
    96‚Üí    96‚Üídevice credentials for edge telemetry, and CSRF protection on cookie-authenticated mutations.
    97‚Üí    97‚Üí
    98‚Üí    98‚Üí### 2.3 SQLite as the sole data store, with per-country fan-out
    99‚Üí    99‚Üí
   100‚Üí   100‚ÜíThe persistence layer is raw `sqlite3` (stdlib), no ORM, no async driver. Worse, it maintains
   101‚Üí   101‚Üí**five separate SQLite files** ‚Äî one per country (PH, TH, VN, MY, ID) ‚Äî and almost every read
   102‚Üí   102‚Üíoperation iterates all five:
   103‚Üí   103‚Üí
   104‚Üí   104‚Üí- `list_vehicles()`, `list_incidents()`, `load_alerts()`, `load_vehicle_states()`,
   105‚Üí   105‚Üí  `has_open_alert()`, `has_recent_vehicle_alert()`, `country_for_route()`, `route_exists()`,
   106‚Üí   106‚Üí  `delete_vehicle()`, `load_route_polyline()`, `database_status()` all do
   107‚Üí   107‚Üí  `for code in COUNTRY_CODES: ... query ...` then merge in Python.
   108‚Üí   108‚Üí- `country_for_route()` is called on **every telemetry write** (`save_vehicle_state`), every
   109‚Üí   109‚Üí  alert save, every chat save ‚Äî that's 5 `SELECT`s per write.
   110‚Üí   110‚Üí- `has_open_alert()` and `has_recent_vehicle_alert()` iterate all 5 DBs per alert check, and
   111‚Üí   111‚Üí  these are called on **every** telemetry upsert (`state.py:196, 231`).
   112‚Üí   112‚Üí- `database_status()` runs ~10 aggregate queries √ó 5 countries = **50 queries per call**.
   113‚Üí   113‚Üí
   114‚Üí   114‚ÜíAdditionally, module-level shared state is accessed across threads without locking:
   115‚Üí   115‚Üí`_connections` dict, `_LAST_TELEMETRY_LOG_WRITE` dict (read-modify-write race), and
   116‚Üí   116‚Üí`_ROUTE_CACHE` are all touched by both the Uvicorn request threadpool and the demo simulator
   117‚Üí   117‚Üídaemon thread.
   118‚Üí   118‚Üí
   119‚Üí   119‚Üí**Remediation:** PostgreSQL as the primary OLTP store with a proper connection pool
   120‚Üí   120‚Üí(asyncpg/SQLAlchemy async). Multi-tenancy via a `country_code` column + indexes + row-level
   121‚Üí   121‚Üísecurity, not file-per-country. Redis for ephemeral live-fleet state and caching. SQLite
   122‚Üí   122‚Üíremains only as an edge-device local buffer.
   123‚Üí   123‚Üí
   124‚Üí   124‚Üí### 2.4 No service layer ‚Äî business logic in route handlers
   125‚Üí   125‚Üí
   126‚Üí   126‚Üí`api/routes.py` contains ~740 lines with inline business logic: `parse_route_file`,
   127‚Üí   127‚Üí`parse_geojson_routes`, `parse_csv_routes`, `parse_gtfs_routes`, `validate_polyline`,
   128‚Üí   128‚Üí`validate_imported_routes`, `simplify_polyline` (Ramer-Douglas-Peucker). The `get_fleet`
   129‚Üí   129‚Üíhandler merges DB vehicles with live state directly in the handler function. There is no
   130‚Üí   130‚Üíservice / repository separation.
   131‚Üí   131‚Üí
   132‚Üí   132‚Üí`core/transit.py` is a **1,860-line god-module** mixing: place search, geocoding, route
   133‚Üí   133‚Üímatching, multi-leg routing, vehicle ranking, fare estimation, NLP pattern matching,
   134‚Üí   134‚Üítranslation, fuzzy matching, and language detection. Massive single-responsibility violation.
   135‚Üí   135‚Üí
   136‚Üí   136‚Üí**Remediation:** Layered architecture: routes ‚Üí services ‚Üí repositories ‚Üí models. `transit.py`
   137‚Üí   137‚Üísplits into `geocoding`, `route_matching`, `multi_leg`, `fare`, `nlp`, `i18n` modules. Each
   138‚Üí   138‚Üíservice has an interface and is unit-testable in isolation.
   139‚Üí   139‚Üí
   140‚Üí   140‚Üí### 2.5 No input validation beyond type checks
   141‚Üí   141‚Üí
   142‚Üí   142‚ÜíPydantic models validate types but almost no constraints:
   143‚Üí   143‚Üí
   144‚Üí   144‚Üí- `Telemetry.latitude` / `longitude` are bare `float` ‚Äî a client can send `lat=999`.
   145‚Üí   145‚Üí- `occupancy: int` has no `ge=0`.
   146‚Üí   146‚Üí- `timestamp: str` accepts any string (should be `datetime`).
   147‚Üí   147‚Üí- `vehicle_id`, `route`, `driver` ‚Äî `str` with no length limit.
   148‚Üí   148‚Üí- `ChatQuery.history: Optional[list[dict]]` ‚Äî no schema for the dicts.
   149‚Üí   149‚Üí- `SuggestionQuery.limit: int = 5` ‚Äî no upper bound (could be 10,000,000).
   150‚Üí   150‚Üí- `POST /routes/import` reads the entire upload into memory with no size limit ‚Äî DoS vector.
   151‚Üí   151‚Üí
   152‚Üí   152‚Üí**Remediation:** Every Pydantic model uses `Field(..., ge=, le=, min_length=, max_length=,
   153‚Üí   153‚Üípattern=)` constraints. File uploads are streaming-parsed with a hard size cap. A shared
   154‚Üí   154‚Üí`validators` module centralizes reusable constraints (lat/lon bounds, ISO timestamps, vehicle
   155‚Üí   155‚ÜíID patterns).
   156‚Üí   156‚Üí
   157‚Üí   157‚Üí### 2.6 No API versioning, no response models, no OpenAPI discipline
   158‚Üí   158‚Üí
   159‚Üí   159‚ÜíAll endpoints live under `/api` with no version prefix. No `response_model` on any endpoint,
   160‚Üí   160‚Üíso the auto-generated OpenAPI schema has no response definitions. No `tags`, so `/docs` is a
   161‚Üí   161‚Üíflat list. No `responses=` to document error codes. The README documents endpoints that
   162‚Üí   162‚Üí**do not exist**: `/api/demand/forecast`, `/api/eta/{vehicle_id}`,
   163‚Üí   163‚Üí`/api/alerts/{id}/false-alarm`, `/api/alerts/{id}/escalate`, `/ws/fleet`.
   164‚Üí   164‚Üí
   165‚Üí   165‚Üí**Remediation:** `/api/v1/` prefix. Every endpoint declares `response_model`, `tags`,
   166‚Üí   166‚Üí`summary`, `description`, and `responses=`. OpenAPI spec is exported, committed, and diffed in
   167‚Üí   167‚ÜíCI ‚Äî breaking schema changes fail the build. README is generated from the spec, not hand-
   168‚Üí   168‚Üíwritten.
   169‚Üí   169‚Üí
   170‚Üí   170‚Üí### 2.7 Hardcoded values and magic numbers (partial list)
   171‚Üí   171‚Üí
   172‚Üí   172‚Üí| Location | Value | Problem |
   173‚Üí   173‚Üí|---|---|---|
   174‚Üí   174‚Üí| `chatbot.py:106` | `"gemini-2.5-flash-lite"` | Model name hardcoded |
   175‚Üí   175‚Üí| `ollama_chatbot.py:10-11` | `OLLAMA_API_URL = "http://localhost:11434"`, `MODEL_NAME = "llama3.2:3b"` | Won't work in container |
   176‚Üí   176‚Üí| `routes.py:107` | `route="04L"`, `time_of_day=8.0`, `traffic_factor=1.0` | PH-specific defaults leak into other countries |
   177‚Üí   177‚Üí| `state.py:246` | `self._alerts = self._alerts[-100:]` | Magic 100 cap |
   178‚Üí   178‚Üí| `state.py:249-255` | traffic-factor weights `0.9, 1.05, 1.2, 1.35` | Magic |
   179‚Üí   179‚Üí| `sqlite_store.py:22` | `TELEMETRY_LOG_INTERVAL_SECONDS = 15` | Magic |
   180‚Üí   180‚Üí| `transit.py:13-27` | 9 magic constants (radii, speeds, ratios, TTLs) | Not in config |
   181‚Üí   181‚Üí| `transit.py:373` | `urlopen(request, timeout=1.8)` | 1.8 s ‚Äî unusual |

   186‚Üí   186‚Üí
   187‚Üí   187‚Üí**Remediation:** All thresholds, fares, model names, timeouts, and radii move into a typed
   188‚Üí   188‚Üíconfig system (pydantic-settings) with per-country overrides. Seed data is culturally neutral
   189‚Üí   189‚Üíor per-country appropriate.
   190‚Üí   190‚Üí
   191‚Üí   191‚Üí### 2.8 DEMO_MODE defaults to true
   192‚Üí   192‚Üí
   193‚Üí   193‚Üí`is_demo_mode()` (`config.py:58-61`) reads the `DEMO_MODE` env var and **defaults to "true"**.
   194‚Üí   194‚ÜíThis means a production deployment, unless someone explicitly sets `DEMO_MODE=false`, will run
   195‚Üí   195‚Üíthe synthetic fleet simulator ‚Äî a daemon thread that pollutes `telemetry_logs` with fake
   196‚Üí   196‚Üívehicle data every 3 seconds. The demo simulator lives inside `core/` (the production
   197‚Üí   197‚Üípackage), not in `dev/` or `tools/`.
   198‚Üí   198‚Üí
   199‚Üí   199‚Üí**Remediation:** `DEMO_MODE` defaults to `false`. The simulator moves to `dev/simulator/` and
   200‚Üí   200‚Üíis launched explicitly. Production images do not contain simulator code.
   201‚Üí   201‚Üí
   202‚Üí   202‚Üí### 2.9 No structured logging, no metrics, no tracing
   203‚Üí   203‚Üí
   204‚Üí   204‚ÜíThe entire backend uses `print()` for logging (`chatbot.py:137`, `ollama_chatbot.py:211`,
   205‚Üí   205‚Üí`sqlite_store.py:580`). No `logging` module, no structlog/loguru, no log levels, no JSON log
   206‚Üí   206‚Üíformat. No Prometheus metrics endpoint. No OpenTelemetry tracing. No Sentry. The `/health`
   207‚Üí   207‚Üíendpoint returns `{"status": "ok"}` with no DB check, no model-load check, no dependency
   208‚Üí   208‚Üícheck ‚Äî no liveness/readiness split.
   209‚Üí   209‚Üí
   210‚Üí   210‚Üí**Remediation:** structlog with JSON output and correlation IDs. Prometheus `/metrics`
   211‚Üí   211‚Üíendpoint with RED metrics (rate, errors, duration) per route. OpenTelemetry traces with
   212‚Üí   212‚Üíauto-instrumentation for FastAPI, asyncpg, and HTTP clients. Sentry for error tracking.
   213‚Üí   213‚Üí`/health` (liveness) and `/ready` (readiness) split, with readiness checking DB + cache +
   214‚Üí   214‚Üímodel availability.
   215‚Üí   215‚Üí
   216‚Üí   216‚Üí### 2.10 No rate limiting
   217‚Üí   217‚Üí
   218‚Üí   218‚ÜíNo `slowapi` or equivalent. `/chatbot` could trigger expensive NLP regex matching (or LLM
   219‚Üí   219‚Üícalls if wired). `/places` makes external HTTP calls to Photon with a 1.8 s timeout ‚Äî easy to
   220‚Üí   220‚Üíabuse. `/routes/import` parses uploaded files with no size limit.
   221‚Üí   221‚Üí
   222‚Üí   222‚Üí**Remediation:** Token-bucket rate limiting per IP and per API key, configurable per route.
   223‚Üí   223‚ÜíThe chatbot and places endpoints get stricter limits. A middleware-based limiter backed by
   224‚Üí   224‚ÜíRedis so it works across multiple workers.

   228‚Üí   228‚Üí## 3. Frontend failures
   229‚Üí   229‚Üí
   230‚Üí   230‚ÜíThe frontend (`app/`) is vanilla HTML/CSS/JS ‚Äî three static HTML pages (`index.html`,
   231‚Üí   231‚Üí`mobile.html`, `operator.html`), seven CSS files, eight JS files totaling ~3,605 lines, plus
   232‚Üí   232‚Üí~181 KB of vendored Leaflet committed to the repo. No build step, no module system, no type
   233‚Üí   233‚Üísafety, no framework, no tests.
   234‚Üí   234‚Üí
   235‚Üí   235‚Üí### 3.1 No build step and no module system
   236‚Üí   236‚Üí
   237‚Üí   237‚ÜíEight JS files served as raw text via classic `<script src>` tags. All top-level function
   238‚Üí   238‚Üídeclarations become implicit `window` globals. Load order is critical and undocumented. A
   239‚Üí   239‚Üífunction defined in `routes-admin.js` silently overrides the same-named function in `map.js`
   240‚Üí   240‚Üí(both define `setupRecenterButtons`, `fitRoute`, `fitFleet`, `previewRoute`, `zoomVehicle`).
   241‚Üí   241‚ÜíThe `operator.js` version of `renderRouteDirectory` references an **undefined variable
   242‚Üí   242‚Üí`groupBy`** (`operator.js:282, 285`) ‚Äî it would throw `ReferenceError` if ever called, but
   243‚Üí   243‚Üíit's dead-but-overriding code from copy-paste.
   244‚Üí   244‚Üí
   245‚Üí   245‚Üí**Remediation:** TypeScript + Vite. ES modules with explicit imports/exports. Build step
   246‚Üí   246‚Üígives minification, tree-shaking, code-splitting, and source maps. Load-order bugs become
   247‚Üí   247‚Üíimpossible.
   248‚Üí   248‚Üí
   249‚Üí   249‚Üí### 3.2 XSS risks via innerHTML and inline onclick
   250‚Üí   250‚Üí
   251‚Üí   251‚Üí~50+ `innerHTML` assignments. Most use `escapeHtml()`, but `escapeHtml` does **not** escape
   252‚Üí   252‚Üíbackticks or `$`, so interpolating its output into JS template literals is unsafe. Map popups
   253‚Üí   253‚Üí(`map.js:197, 209`) use inline `onclick="showVehicleDetailsModal('${escapeHtml(...)}')"` ‚Äî
   254‚Üí   254‚Üíinside an HTML attribute, entities are decoded, so an apostrophe in `vehicle_id` breaks the JS
   255‚Üí   255‚Üístring syntax. `mobile.js:127, 177` interpolate `route` and `vehicle_id` into `innerHTML`
   256‚Üí   256‚Üí**without** `escapeHtml` at all ‚Äî currently benign only because the simulator uses safe
   257‚Üí   257‚Üístrings.

   260‚Üí   260‚Üíauto-escaped. No `dangerouslySetInnerHTML` without DOMPurify sanitization. No inline event

   340‚Üí   340‚Üí## 4. Edge-layer integrity failures
   341‚Üí   341‚Üí
   342‚Üí   342‚ÜíThis is the most serious category. The original LoadSense **pitched YOLOv8-nano computer
   343‚Üí   343‚Üívision** but the edge code does not do computer vision.
   344‚Üí   344‚Üí
   345‚Üí   345‚Üí### 4.1 `line_crossing_counter.py` `webcam`/`video` modes are fake CV
   346‚Üí   346‚Üí
   347‚Üí   347‚ÜíThe file has three modes via `--source`:
   348‚Üí   348‚Üí
   349‚Üí   349‚Üí- `sim` (default): pure random-walk in pixel space. Honest, labeled as simulation.
   350‚Üí   350‚Üí- `webcam` / `video`: `import cv2`, `cv2.VideoCapture(source)`, reads frames in a loop ‚Äî
   351‚Üí   351‚Üí  **then ignores every pixel of every frame**. The only frame-derived value is
   352‚Üí   352‚Üí  `gray_mean = int(frame.mean())` (line 103), and `movement = ((gray_mean % 17) - 8)` (line
   353‚Üí   353‚Üí  104). **No object detection, no tracking, no YOLO, no DeepSORT/ByteTrack.** Frame brightness
   354‚Üí   354‚Üí  mod 17 is treated as "person movement".
   355‚Üí   355‚Üí
   356‚Üí   356‚ÜíYet `docs/loadsense_pitch_outline.md` Section 5A explicitly states: *"We run a lightweight
   357‚Üí   357‚Üícomputer vision model‚Äîspecifically object detection like YOLO‚Äîto track passengers crossing
   358‚Üí   358‚Üíthe door threshold."* The code does not deliver this; the pitch does. This is an **integrity
   359‚Üí   359‚Üígap**, not merely a technical debt item.
   360‚Üí   360‚Üí
   361‚Üí   361‚Üí**Remediation:** Re-LoadSense is honest. The edge layer is either (a) real YOLOv8-nano via
   362‚Üí   362‚ÜíUltralytics running on a Raspberry Pi 5 / Jetson Nano, with ByteTrack for identity tracking,
   363‚Üí   363‚Üíor (b) a clearly-labeled `sim` mode that emits the same contract. There is no `webcam` mode
   364‚Üí   364‚Üíthat opens a camera and ignores pixels. The contract between edge and cloud is a versioned
   365‚Üí   365‚Üíprotobuf/JSON Schema, not ad-hoc.
   366‚Üí   366‚Üí
   367‚Üí   367‚Üí### 4.2 No edge-cloud contract schema
   368‚Üí   368‚Üí
   369‚Üí   369‚ÜíThe telemetry payload is ad-hoc JSON with no formal schema, no version field, no device
   370‚Üí   370‚Üíidentity, no firmware version, no auth token, no heartbeat. `mock_telemetry.py` and
   371‚Üí   371‚Üí`line_crossing_counter.py` emit slightly different field sets. No protobuf, no Avro, no JSON
   372‚Üí   372‚ÜíSchema, no AsyncAPI.
   373‚Üí   373‚Üí
   374‚Üí   374‚Üí**Remediation:** Versioned edge-cloud contract (protobuf or JSON Schema with a `schema_version`
   375‚Üí   375‚Üífield). AsyncAPI spec for the telemetry channel. Device identity, firmware version, and auth
   376‚Üí   376‚Üítoken in every payload. Heartbeat/liveness separate from telemetry.
   377‚Üí   377‚Üí
   378‚Üí   378‚Üí### 4.3 No offline buffer, no device management, no OTA
   379‚Üí   379‚Üí
   380‚Üí   380‚Üí- HTTP failure on the edge = data loss. `post_telemetry` prints to stderr and drops the row.
   381‚Üí   381‚Üí- No device registry (which Pi is on which vehicle).
   382‚Üí   382‚Üí- No device provisioning flow, no device-to-vehicle binding API.
   383‚Üí   383‚Üí- No OTA firmware updates, no A/B partition, no rollback.
   384‚Üí   384‚Üí- No edge ML lifecycle ‚Äî no model download from cloud to edge, no versioned edge model, no
   385‚Üí   385‚Üí  edge inference metric reporting.
   386‚Üí   386‚Üí- No edge auth ‚Äî no API key, no mTLS, no JWT, no per-device credentials.
   387‚Üí   387‚Üí- No edge runtime ‚Äî no container image, no systemd unit, no K3s/balena.
   388‚Üí   388‚Üí
   389‚Üí   389‚Üí**Remediation:** A real edge platform: device registry + provisioning API, mTLS device
   390‚Üí   390‚Üícertificates, offline buffer (SQLite/RocksDB queue on device), batched compressed upload with
   391‚Üí   391‚Üíretry/backoff, OTA firmware pipeline with A/B partitions and automatic rollback, edge model
   392‚Üí   392‚Üídownload with version negotiation, edge inference metrics reported to cloud, containerized
   393‚Üí   393‚Üíedge runtime (balena/balenaCloud or K3s).
   394‚Üí   394‚Üí
   395‚Üí   395‚Üí### 4.4 Privacy leakage pattern
   396‚Üí   396‚Üí
   397‚Üí   397‚ÜíThe pitch claims "no video leaves the device" but `line_crossing_counter.py` writes
   398‚Üí   398‚Üí`centroid_x`, `centroid_y`, `density_zone` per frame to a CSV committed to the repo. This is
   399‚Üí   399‚Üía metadata leakage pattern that real YOLO bbox data would inherit unless explicitly redacted.
   400‚Üí   400‚Üí
   401‚Üí   401‚Üí**Remediation:** Edge devices emit only aggregate counts (boarded, alighted, current
   402‚Üí   402‚Üíoccupancy, density-zone histogram). No per-frame coordinates, no bboxes, no images ever leave
   403‚Üí   403‚Üíthe device. A privacy impact assessment (PIA) is written and reviewed before any pilot.
   404‚Üí   404‚Üí

   407‚Üí   407‚Üí## 5. ML / MLOps failures
   408‚Üí   408‚Üí
   409‚Üí   409‚ÜíThe ML layer (`cloud/` + `backend/app/core/phase2.py`) is broken end-to-end.
   410‚Üí   410‚Üí
   411‚Üí   411‚Üí### 5.1 Trained artifacts do not exist
   412‚Üí   412‚Üí
   413‚Üí   413‚ÜíREADME, `phase-2.md`, `cloud/README.md`, and `DATA_SOURCES_AND_APIS.md` all claim "checked-in
   414‚Üí   414‚Üíartifacts" but `cloud/artifacts/` **does not exist in the repo**. `phase2.py:load_eta_model()`
   415‚Üí   415‚Üíreturns `None` (line 20) and `predict_eta_details` silently falls back to a config-driven
   416‚Üí   416‚Üíheuristic returning `{"source": "fallback"}`. The trained model is effectively dead code in
   417‚Üí   417‚Üíproduction.
   418‚Üí   418‚Üí
   419‚Üí   419‚Üí### 5.2 Train/serve path mismatch for demand forecast
   420‚Üí   420‚Üí
   421‚Üí   421‚Üí`train_demand_forecast.py` writes to `cloud/artifacts/demand_forecast.json` (single file).
   422‚Üí   422‚Üí`phase2.py:60-61` reads `cloud/artifacts/demand/{COUNTRY}_demand_forecast.json` (5 per-country
   423‚Üí   423‚Üífiles). **They never connect.** Even if you run the trainer successfully, the backend will
   424‚Üí   424‚Üínever pick up the output.
   425‚Üí   425‚Üí
   426‚Üí   426‚Üí### 5.3 Target leakage in ETA training
   427‚Üí   427‚Üí
   428‚Üí   428‚Üí`train_eta_model.py:build_target()` (lines 23-25) constructs the target as a deterministic
   429‚Üí   429‚Üílinear function of three of the four input features:
   430‚Üí   430‚Üí
   431‚Üí   431‚Üí```python
   432‚Üí   432‚Üínoise = (frame["stop_index"] % 3) * 0.2
   433‚Üí   433‚Üíreturn 4.5 + frame["stop_index"] * 0.8 + frame["time_of_day"] * 0.06
   434‚Üí   434‚Üí            + frame["traffic_factor"] * 1.9 + frame["count"] * 0.08 + noise
   435‚Üí   435‚Üí```
   436‚Üí   436‚Üí
   437‚Üí   437‚ÜíThe GradientBoostingRegressor trivially memorizes the linear coefficients and reports an MAE
   438‚Üí   438‚Üínear the noise floor (~0.2 minutes). The reported evaluation is **meaningless**. No held-out
   439‚Üí   439‚Üíroute, no held-out time period, no k-fold CV, no baseline comparison.
   440‚Üí   440‚Üí
   441‚Üí   441‚Üí### 5.4 Non-deterministic demand forecast in production
   442‚Üí   442‚Üí
   443‚Üí   443‚Üí`phase2.py:118` (the no-artifact fallback) uses `random.uniform(-0.5, 0.5)` without a seed.
   444‚Üí   444‚ÜíEvery call to `GET /api/demand` returns **different numbers** for the same route and hour. The
   445‚Üí   445‚Üí`model` field is `"dynamic_simulation"` but the API gives no indication the values are non-
   446‚Üí   446‚Üídeterministic. Operators would see the dashboard "jitter" between refreshes.
   447‚Üí   447‚Üí
   448‚Üí   448‚Üí### 5.5 `pickle.load` is an RCE vector
   449‚Üí   449‚Üí
   450‚Üí   450‚Üí`phase2.py:19` does `pickle.load(handle)` on `cloud/artifacts/eta_model.pkl`. `pickle.load`
   451‚Üí   451‚Üíexecutes arbitrary code from the file. Combined with `CORS allow_origins=["*"]` and no auth,
   452‚Üí   452‚Üíthis is a critical-severity issue: if an attacker can write the `.pkl` (path traversal,
   453‚Üí   453‚Üísupply-chain compromise, misconfigured CI), they get remote code execution on every `/api/eta`
   454‚Üí   454‚Üícall.
   455‚Üí   455‚Üí
   456‚Üí   456‚Üí### 5.6 Heavy deps for dead code
   457‚Üí   457‚Üí
   458‚Üí   458‚Üí`requirements.txt` ships `xgboost>=2.0`, `prophet>=1.1`, `google-genai>=0.3.0`. Grep confirms
   459‚Üí   459‚Üí**xgboost is never imported** (only sklearn's `GradientBoostingRegressor` is used). `prophet`
   460‚Üí   460‚Üí(~200 MB with cmdstanpy) is used by a trainer whose output is never wired into production.
   461‚Üí   461‚Üí`google-genai` is consumed only by dead `chatbot.py`. These bloat the Docker image by hundreds
   462‚Üí   462‚Üíof MB for no benefit.
   463‚Üí   463‚Üí
   464‚Üí   464‚Üí### 5.7 No MLOps whatsoever
   465‚Üí   465‚Üí
   466‚Üí   466‚ÜíNo model registry (MLflow/W&B), no experiment tracking, no feature store (train-serve skew is
   467‚Üí   467‚Üístructurally guaranteed), no data validation (Great Expectations/Pandera), no drift detection
   468‚Üí   468‚Üí(evidently/alibi-detect), no production monitoring (no Prometheus metrics on `/api/eta` or
   469‚Üí   469‚Üí`/api/demand`), no A/B testing, no canary, no CI/CD for ML, no model serving infrastructure
   470‚Üí   470‚Üí(no Triton/BentoML/TorchServe). The "MLOps" is: developer runs a script locally, commits a
   471‚Üí   471‚Üí`.pkl`, FastAPI loads it via `@lru_cache`.
   472‚Üí   472‚Üí
   473‚Üí   473‚Üí### 5.8 Synthetic data is unrealistic and unseeded
   474‚Üí   474‚Üí
   475‚Üí   475‚Üí`data/generate_synthetic_history.py` produces 500 rows, 15-min intervals, 4 routes cycled
   476‚Üí   476‚Üí(only 4 of 74 PH routes), `count` as a linear function of `stop_index` and `traffic_factor`.
   477‚Üí   477‚Üí`random` is **unseeded** ‚Äî every run produces a different CSV. No day-of-week, no holiday
   478‚Üí   478‚Üíeffects, no weather, no events, no spatial correlation. The committed snapshot is one
   479‚Üí   479‚Üíarbitrary realization.
   480‚Üí   480‚Üí
   481‚Üí   481‚Üí**Remediation (all of ¬ß5):** Real MLOps stack ‚Äî MLflow for registry + tracking, Feast (or
   482‚Üí   482‚Üíinline typed feature definitions) for feature parity, evidently for drift detection,
   483‚Üí   483‚ÜíPrometheus metrics on inference endpoints, joblib/ONNX for safe model serialization (never
   484‚Üí   484‚Üípickle for untrusted paths), seeded reproducible data generation, train/serve schema
   485‚Üí   485‚Üívalidation, champion/challenger promotion pipeline, CI/CD that retrains on data refresh and
   486‚Üí   486‚Üípromotes models through dev ‚Üí staging ‚Üí prod with automated rollback.

   561‚Üí   561‚Üí## 8. Security & privacy failures
   562‚Üí   562‚Üí
   563‚Üí   563‚ÜíConsolidated from the sections above:
   564‚Üí   564‚Üí
   565‚Üí   565‚Üí1. **CORS `allow_origins=["*"]`** (`main.py:36`) ‚Äî fully open to any website.

   567‚Üí   567‚Üí3. **No rate limiting** anywhere.
   568‚Üí   568‚Üí4. **`pickle.load` RCE** (`phase2.py:19`).
   569‚Üí   569‚Üí5. **Information disclosure** via `str(exc)` in `HTTPException(500, ...)` (6+ handlers) and
   570‚Üí   570‚Üí   WebSocket error messages (`routes.py:101`).
   571‚Üí   571‚Üí6. **No input size limits** on `/routes/import`.
   572‚Üí   572‚Üí7. **No PII redaction** in `chatbot_queries` persistence; chat history (which may contain
   573‚Üí   573‚Üí   phone numbers and addresses) is sent on every subsequent chat request.
   574‚Üí   574‚Üí8. **Generic User-Agent** for Nominatim/Photon ‚Äî violates Nominatim usage policy.
   575‚Üí   575‚Üí9. **`DEMO_MODE` defaults to true** ‚Äî production runs the synthetic fleet simulator.
   576‚Üí   576‚Üí10. **No HTTPS enforcement**, no HSTS, no secure cookies.
   577‚Üí   577‚Üí11. **`reset_database`** is unauthenticated, irreversible, AND seeds fake vehicles with
   578‚Üí   578‚Üí    hardcoded culturally-specific names.
   579‚Üí   579‚Üí12. **No CSRF protection** for state-changing endpoints.
   580‚Üí   580‚Üí13. **No security headers** (CSP, X-Frame-Options, X-Content-Type-Options).
   581‚Üí   581‚Üí14. **No dependency audit** ‚Äî `>=` pins, no `pip-audit`/`safety`/SBOM.
   582‚Üí   582‚Üí15. **No privacy impact assessment** despite passenger-counting from cameras. No Philippines
   583‚Üí   583‚Üí    Data Privacy Act 2012 compliance review. No consent flow. No retention policy. No data
   584‚Üí   584‚Üí    subject rights mechanism.
   585‚Üí   585‚Üí16. **Edge metadata leakage** ‚Äî per-frame coordinates written to CSV.
   586‚Üí   586‚Üí
   587‚Üí   587‚Üí**Remediation:** Security is a first-class concern in Re-LoadSense, not a phase-2 item. See
   588‚Üí   588‚Üí`02-improved-concept.md` ¬ß9 (Security) and ¬ß10 (Privacy & Compliance) for the full model:
   589‚Üí   589‚Üíauthn/authz, RBAC, mTLS for edge devices, rate limiting, CSP, dependency scanning, SAST/DAST,
   590‚Üí   590‚Üíthreat model (STRIDE), PIA, DPA 2012 compliance, data retention, data subject rights.

   594‚Üí   594‚Üí## 9. Operability & DevOps failures
   595‚Üí   595‚Üí
   596‚Üí   596‚Üí1. **No CI/CD** ‚Äî no `.github/workflows/`, no `Makefile`, no CI config anywhere.
   597‚Üí   597‚Üí2. **No health checks** in `docker-compose.yml`.
   598‚Üí   598‚Üí3. **No monitoring** ‚Äî no Prometheus, no structured logs, no dashboards.
   599‚Üí   599‚Üí4. **No graceful shutdown** ‚Äî `lifespan` stops the simulator but doesn't close DB connections.
   600‚Üí   600‚Üí5. **No DB backups** ‚Äî SQLite files on local disk, no scheduled backups.
   601‚Üí   601‚Üí6. **No secrets management** ‚Äî `.env` file expected, no vault.
   602‚Üí   602‚Üí7. **Single-stage Dockerfile** ‚Äî no multi-stage build, runs as root, `COPY . .` copies the
   603‚Üí   603‚Üí   entire repo including `docs/`, `tests/`, `.git/`.
   604‚Üí   604‚Üí8. **Uvicorn runs as root** with 1 worker, no `--workers`.
   605‚Üí   605‚Üí9. **No pinned dependencies** ‚Äî `>=` everywhere, non-reproducible builds.
   606‚Üí   606‚Üí10. **Heavy deps for dead code** ‚Äî prophet, xgboost, google-genai bloat the image.
   607‚Üí   607‚Üí11. **Tests are scripts** ‚Äî not discoverable by pytest, no CI, write to real DB.
   608‚Üí   608‚Üí12. **No staging environment**, no prod deployment guide.
   609‚Üí   609‚Üí13. **No load testing**, no operational runbooks, no on-call, no incident response docs.
   610‚Üí   610‚Üí
   611‚Üí   611‚Üí**Remediation:** Multi-stage Dockerfile, non-root user, distroless or slim base, pinned deps
   612‚Üí   612‚Üíwith hashes. GitHub Actions CI (lint, type-check, test, build, security scan, docs drift
   613‚Üí   613‚Üícheck). Docker Compose with healthchecks for dev; Helm chart / k8s manifests for prod.
   614‚Üí   614‚ÜíPrometheus + Grafana for metrics, Loki for logs, OpenTelemetry for traces. Runbooks per
   615‚Üí   615‚Üíservice. SLO/SLI definitions with error budgets.

   692‚Üí   692‚Üíthe most integrity-critical fixes (honest edge, real auth, reproducible ML) land first.
